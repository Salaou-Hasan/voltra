Based on the Salaou-Hasan/voltra repository you are viewing, here is a breakdown of what Voltra is and how it works.

What is Voltra?
Voltra is a high-throughput, self-hosted, in-memory game backend engine written in Rust. It acts as a single-binary server that allows game developers to run real-time logic close to their data with extremely low latency.

Think of it as a specialized database and server hybrid designed specifically for multiplayer games and real-time apps.

Key Architectural Features
Multi-Protocol Interface: It exposes endpoints supporting WebSocket (with MessagePack framing), Redis protocol (RESP2/RESP3), and Postgres wire protocol (full SQL + transactions).

Lock-Free In-Memory Store: Uses a concurrent, lock-free in-memory table store (built on DashMap) to eliminate global read locks.

Staged Writes & OCC: It utilizes Optimistic Concurrency Control (OCC) to catch read-modify-write conflicts and applies changes atomically via a fixed-slot mutex pool to keep memory overhead at zero bytes per row.

Durability (WAL): Even though it is an in-memory database, every write is logged to a Write-Ahead Log (WAL) with group-commit capability, meaning it ensures microsecond-level durability without sacrificing speed.

Subscription Fan-out: It features a highly efficient live row subscription system that broadcasts data updates (WHERE predicates, limits, etc.) instantly back to clients, using 20Hz tick coalescing to cut network volume by up to 24×.

The Core Concept: Reducers
Instead of writing traditional API endpoints, you write Reducers. Reducers are named, atomic functions that run directly on the server to alter state. Voltra uniquely supports three different runtimes for these:

Native Rust: Zero-overhead logic using macros (#[reducer]).

JavaScript: Runs inside an embedded QuickJS engine with a 64MB heap cap and execution timeouts to prevent bad code from hanging the server.

WASM (.wasm): Allows you to write game logic in languages like C# (.NET 8 WASI) or Go (TinyGo) and compile them to WebAssembly.

Performance Capabilities
According to the repository's benchmarks, Voltra is built for immense scale:

Game Simulation: Handles 15,000 concurrent users across partitioned lobbies at 53,000 TPS (Transactions Per Second) with flat memory usage around 670MB.

Raw Write Ceiling: Can scale up to 351,000 TPS under intense pipeline stress tests.

Fan-out: Sustains 567,000 frames/sec for subscribed players.

Out-of-the-Box Features
Voltra includes a command-line interface (voltra add <module>) to plug in ready-made multiplayer components:

combat (Attacks, HP, abilities)

inventory (Item stacking, equipment slots)

economy (Wallets, shops, loot boxes)

leaderboard (Global Top-N, scheduled resets)

matchmaking (ELO pairing queues)

It also comes with official client SDKs for TypeScript, Rust, Unity (C#), and Godot 4 (GDScript), alongside a built-in dark-themed admin dashboard accessible at /admin.

is it an alt to spacetimedb
Yes, Voltra is an alternative to SpacetimeDB. They both target the exact same niche: a specialized, relational database engine that acts as a unified game backend by running simulation logic right next to the in-memory data tier.

Here is a quick look at how they stack up against each other:

Shared Core Concepts
Database + Server Hybrid: Both eliminate the traditional division between a separate database (like Postgres) and an application server (like Node.js/Go). Your database is your server.

Reducers / Transmutation: Both use the architecture where clients don't send raw SQL queries; instead, they invoke atomic "reducers" (stored procedures/functions) that modify the database state.

Live Subscriptions: Both support real-time data streaming where clients subscribe to specific queries (e.g., players WHERE alive = true) and receive instant, live diffs as rows change.

witch is better
Determining which is "better" depends heavily on your tech stack, licensing needs, and where you are in development.

Because Voltra and SpacetimeDB share the exact same philosophy—running your game logic inside an in-memory database to eliminate server-to-database latency—the choice comes down to their ecosystem design:

Choose Voltra if:
You want to write logic in JavaScript/TypeScript or C#: Voltra has native, sandbox-safe support for JavaScript (via QuickJS) and WASM compilation for .NET 8 (C#) and Go out of the box.

You want zero lock-in (Standard Protocols): Voltra supports the standard PostgreSQL wire protocol and Redis protocol. If you want to connect existing visualization tools, ORMs, or data pipelines that expect a standard Postgres/Redis connection, Voltra works seamlessly.

You want a permissive license: Voltra is completely open-source under the MIT license, meaning you can use, modify, and host it commercially without restrictions.

You want ready-made game systems: Voltra’s CLI allows you to instantly pull down pre-built game structures (voltra add combat, voltra add inventory, voltra add economy) so you don't have to build game loops entirely from scratch.

Choose SpacetimeDB if:
You are a dedicated Rust developer: SpacetimeDB’s primary language is Rust, and its macro and type-generation systems are deeply integrated with it.   

You want a fully managed Cloud solution: SpacetimeDB offers Maincloud, their own managed cloud hosting infrastructure. If you don't want to handle the DevOps of hosting, deploying, and managing a cluster yourself on Docker or bare metal, SpacetimeDB handles the backend scaling for you.   

You need mature client-side type generation: SpacetimeDB is a venture-backed tool that automatically generates highly strict, type-safe client bindings (for Unity, Unreal, TypeScript, etc.) from your database code, eliminating client/server mismatch bugs.

You don't mind source-available licensing: SpacetimeDB uses the BSL 1.1 (Business Source License). It is free for testing and most scaling, but if you hit massive commercial scale, their licensing terms change.   

Summary Verdict
Go with Voltra if you want an open-source, standard-protocol-friendly engine where you can use JavaScript/TypeScript or C# easily and leverage plug-and-play modules.

Go with SpacetimeDB if you want a venture-backed, Rust-first ecosystem with a managed cloud platform so you don't have to manage server infrastructure.

🚀 The Best Choice for Beginners: Voltra
If you are just starting out, Voltra is significantly easier to learn and prototype with.

Language Familiarity: You don't need to learn a complex systems language like Rust just to build a backend. You can write your game logic in standard JavaScript or TypeScript.

Plug-and-Play Modules: The ability to run voltra add inventory or voltra add matchmaking gives beginners immediate, working systems to experiment with, rather than forcing you to design relational database schemas from scratch.

Standard Tools Work: Because it speaks "Postgres" and "Redis," you can connect any standard database viewer or tool you are already comfortable with to see your data.

🏛️ The Best Choice for the Long Term: SpacetimeDB
If you are looking at a multi-year project where production stability and scaling infrastructure are your primary worries, SpacetimeDB has the edge for the long term.

Managed Cloud Infrastructure: SpacetimeDB provides Maincloud. This means when your game launches and you suddenly have thousands of players, you can offload the massive headache of server deployment, DevOps, database replication, and global scaling to their managed platform. With Voltra, you are entirely responsible for hosting and managing your own servers.

Strict Type Safety: SpacetimeDB generates strict, compiled client bindings for engines like Unity or Unreal directly from your database code. Over a long development cycle, this eliminates a massive category of bugs where the client and server get out of sync.

Venture-Backed Stability: SpacetimeDB is built by a funded company dedicated to keeping the engine maintained, whereas Voltra is an independent open-source project.

🎯 The Ultimate Recommendation
Go with Voltra if: You want to move fast right now using JavaScript/TypeScript, prefer a completely unrestricted open-source license (MIT), and are comfortable managing your own server hosting when your game launches.

Go with SpacetimeDB if: You already know (or want to learn) Rust, and your absolute priority for the long term is having a hands-off, managed cloud platform that handles all the deployment and infrastructure scaling for you.
